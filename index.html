Task Assignment and Acceptance

  • Rule: The person who assigns the task is responsible for its acceptance after completion.
  • Best Practices:
    • Clearly formulate requirements and expectations when assigning a task.
    • After completing the task, the executor notifies the assigner and provides results for verification.

Merge Request Process

  • Rule: The executor creates a Merge Request and adds the necessary reviewers.
  • Best Practices:
    • In the Merge Request description, specify the purpose of the changes and the affected code areas.
    • Use tagging (@username) to notify reviewers.
    • Prefer using automated notifications over personal messages to avoid overloading communication channels.
